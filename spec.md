Documento de Especificação Técnica (Technical Spec)1. Arquitetura Geral do SistemaO sistema adotará uma arquitetura Cliente-Servidor padrão, onde o cliente (Frontend) será responsável pelo processamento pesado de renderização e interpolação matemática, enquanto o servidor (Backend) atuará como uma API RESTful para persistência de dados e eventuais processamentos de mídia mais complexos.Frontend: Vanilla JavaScript (para performance direta no Canvas), HTML5, Tailwind CSS.Backend: Node.js com Express.js.Banco de Dados: SQLite3 utilizando um ORM leve (como Prisma ou Sequelize) ou queries nativas via sqlite3 para facilitar a portabilidade.2. Especificações do Frontend (Motor Gráfico e Lógica)3.1. Renderização de Canvas (Loop Principal)O canvas utilizará o método requestAnimationFrame para atualizar a tela a 60 FPS durante as animações.Estado Global: Um objeto JavaScript armazenará o estado atual da tela, contendo um array de tokens (jogadores/bola) e lines (desenhos).Redimensionamento e Posicionamento Relativo: As coordenadas armazenadas no JSON não devem ser em pixels absolutos, mas sim em porcentagens (0.0 a 1.0) da largura e altura do campo. Ao renderizar, o sistema multiplica essa porcentagem pelo tamanho real do canvas.3.2. Vínculo da Bola (Snap to Player)Para calcular se a bola entrou na zona de controle (posse) de um jogador, o sistema calculará a distância euclidiana entre o centro da bola e o centro do jogador em cada evento de arrasto (mousemove).A fórmula matemática para detecção de colisão circular é:$$d = \sqrt{(x_2 - x_1)^2 + (y_2 - y_1)^2}$$Se $d < R$ (onde $R$ é o raio de "magnetismo" do jogador, por exemplo, 20 pixels), a propriedade ball.attachedTo recebe o player.id. A partir desse momento, no loop de renderização, as coordenadas da bola serão $x_{bola} = x_{jogador} + offset$.3.3. Interpolação de Keyframes (Animação)Quando o usuário acionar o "Play", o sistema fará a transição entre as coordenadas de um objeto no Frame $A$ para o Frame $B$ com base em uma duração de tempo configurável (ex: 2 segundos).O cálculo utilizará Interpolação Linear (LERP):$$P(t) = P_0 + t(P_1 - P_0)$$Onde:$P(t)$ é a posição atual no tempo.$P_0$ é a posição no Frame inicial.$P_1$ é a posição no Frame de destino.$t$ é o tempo normalizado (variando de 0.0 a 1.0).3.4. Ancoragem de Setas VectoriaisEstrutura da Linha: Uma linha é um objeto contendo { type: 'dashed', originId: 5, targetX: 0.8, targetY: 0.2 }.Atualização em Tempo Real: Se originId existir, o $X_0$ e $Y_0$ da linha serão constantemente sobrescritos pelas coordenadas do jogador ID 5 durante o requestAnimationFrame, fazendo a seta segui-lo.4. Estrutura de API (Endpoints do Backend)O Express.js exporá a seguinte API RESTful para salvar e recuperar dados do SQLite:GET /api/teams - Retorna todos os times e suas cores.GET /api/teams/:id/players - Retorna a lista de jogadores de um time específico.POST /api/plays - Cria um novo projeto de jogada tática.GET /api/plays/:id - Carrega os dados da jogada, incluindo o background escolhido.PUT /api/plays/:id/frames - Salva o array completo de frames (o state_json) associado àquela jogada. Usado no botão de "Salvar Projeto".5. Layout UI (Mapeamento Tailwind CSS)A estrutura DOM seguirá um layout CSS Grid rígido de tela cheia (h-screen w-screen overflow-hidden):Sidebar (Esquerda): <aside class="w-64 bg-gray-800 text-white flex flex-col p-4">Painéis sanfonados para Equipe A e Equipe B.Componentes draggable (jogadores) com o atributo draggable="true".Top Bar (Ferramentas): <header class="h-14 bg-gray-900 flex items-center justify-between px-6">Botões de rádio estlizados para Caneta Contínua, Tracejada e Ondulada.Canvas (Centro): <main class="flex-1 bg-gray-100 flex justify-center items-center relative"><canvas id="tacticalBoard" class="max-w-full max-h-full shadow-lg rounded"></canvas>Bottom Bar (Timeline): <footer class="h-32 bg-gray-800 border-t border-gray-700 p-4">Miniaturas dos frames gerados e barra de reprodução de tempo.6. Lógica de Exportação6.1. Imagem (Estática)No frontend, ao clicar em "Exportar Imagem", o sistema executará:Conclusão da renderização do frame atual.Chamada de canvas.toDataURL('image/png').Criação de um link temporário (<a>) com o atributo download="jogada.png" disparando o download nativo do navegador.6.2. Vídeo (Animado - MP4)Utilizaremos a MediaRecorder API nativa do navegador para evitar carga excessiva no backend:O canvas é capturado como stream de vídeo: const stream = canvas.captureStream(60);.O MediaRecorder grava esse stream.A função de "Play" na linha do tempo é acionada programaticamente.Ao chegar no último frame, a gravação é encerrada, o buffer/blob em formato webm ou mp4 é gerado no navegador e entregue ao usuário via prompt de download.
